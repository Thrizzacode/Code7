1. 簡介

Code7 是一款專為簡化 SVN 合併流程設計的桌面工具。它能自動識別專案路徑，協助開發者快速在 branches、qat 與 stg 三大環境之間進行代碼合併、衝突處理與工作目錄同步。

✨ v1.7.x 新功能： 新增 SVN Log 歷史版本檢視，並在提交頁面支援 Revert (還原) 變更檔案的功能。

2. 快速入門：首次執行

若您是第一次開啟 Code7，系統會引導您進行初始設定：

  1. 點擊 「�� 匯入工作目錄」 按鈕。
  2. 選擇存放所有 Fz 專案的 父資料夾（例如：D:\Workspace\Fz_Projects）。
  3. App 會自動掃描該目錄下所有符合規則的 SVN Working Copy 並匯入專案清單。

3. 核心功能：合併版本 (Merge)

第一步：選擇專案

在頂部導覽列選擇您要作業的專案。選擇後，App 會自動抓取該專案對應的「本地」與「遠端」版本資訊。

第二步：設定來源與目標

  • 來源 (Source)：選擇來源環境（如 branches）及其版本號。
  • 目標 (Target)：選擇目標環境（如 qat）及其對應版本。
  • ✨ 智能同步 (Smart Sync)：如果您選擇的版本在本地尚未下載（Working Copy 不存在），路徑下方會出現 「�� 同步至本地」 按鈕。點擊後系統會自動執行 svn update 並展開目錄（Infinity Depth）。

第三步：執行合併

  1. 選取 Revisions：勾選您想要合併的版本紀錄。
    • 進階過濾：搜尋框支援 正規表達式 (RegExp)。
    • 重複合併 (Cherry-pick)：即使已合併，仍可手動勾選重新合併。
  2. 點擊下方的 「執行合併 (Merge)」。
  3. 後續流程判斷：
    • 無衝突 (Success)：提示成功並自動生成 Commit Message，可直接點擊「提交 (Commit)」完成。
    • 有衝突 (Conflict)：彈出衝突清單，點擊「使用外部工具解決」開啟 TortoiseMerge。

4. 獨立提交功能 (Commit) ✨ NEW

如果您需要單獨處理某個環境的檔案提交（非合併流程），請切換至  「提交 (Commit)」  分頁：

  1. 掃描變更：選擇環境與版本後，App 會自動顯示工作區所有變更檔案（修改、新增、刪除、衝突）。
  2. 檔案預覽：「快點兩下」  檔案或是點擊右側圖示，可直接開啟  TortoiseDiff  比對差異，或是直接開啟該檔案。
  3. 執行提交：勾選要提交的檔案，輸入 Commit Message 後點擊下方按鈕即可。

6. 個人化設定與更新 (Settings)

點擊右上角的 ⚙️ 設定 圖示進入：

  • ✨ IIS 版本切換器：
    • 一鍵整機切換：支援自動同步 hosts 檔案與 applicationHost.config 至各分支版本。
    • 團隊共享配置 (v1.6.1)：新增  「�� 恢復預設」  按鈕，點擊後可一鍵恢復為團隊專屬網路路徑 (\\192.168.70.17...)，支援自動識別最新版本。
    • 排序優化 (v1.6.1)：版本清單現在會依據編號  倒序排列 (最新版本置頂)，方便快速選取。
    • 偵測功能：系統會自動偵測目前 IIS 正在運行的分支版本。
    • 安全性：套用時會彈出 Windows UAC 授權（黑畫面確認），請點選「是」才能寫入系統路徑。
  • 佈景主題切換：提供 PHYSICAM (紅)、TECHNOLOM (藍) 等 5 種風格色彩，並支援  �� 深淺色模式。
  • 專案/工具管理：可手動排除批次同步專案，或指定特定的 TortoiseMerge 路徑。
  • App 更新管理：查看目前的 App 版本並手動檢查更新。

7. 常見問題 (FAQ)

  • Q: 為什麼顯示「載入失敗」？
    • A: 請確保已安裝 SVN CLI 工具並加入環境變數。
  • Q: IIS 切換時為什麼沒反應？
    • A: 請確保您在 UAC 提示時有點擊「是」。若權限不足，系統檔案將無法被覆蓋。
