## ADDED Requirements

### Requirement: SVN 路徑隨選同步 (On-Demand Path Sync)

系統必須提供一種機制，當使用者選取的來源或目標版本僅存在於遠端伺服器時，允許使用者手動將該目錄同步至本地工作副本。

#### Scenario: 成功同步遠端路徑

- **WHEN** 使用者選取了一個標註為 `(遠端)` 的版本，且該路徑在本地不存在。
- **THEN** UI 的路徑預覽區塊應顯示一個「同步至本地」按鈕。
- **WHEN** 使用者點擊「同步至本地」按鈕。
- **THEN** 系統應執行 `svn update --set-depth infinity` 並顯示進度。
- **THEN** 同步成功後，路徑預覽應更新為 `(已就緒)`，且原本的錯誤圖示應消失。

#### Scenario: 同步失敗處理

- **WHEN** SVN 更新指令傳回錯誤（如網路斷線或權限不足）。
- **THEN** 系統應停止載入狀態，並跳出錯誤類型的 Toast 告知使用者具體失敗原因。
- **THEN** 同步按鈕應保持可用，允許使用者重試。
